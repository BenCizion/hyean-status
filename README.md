# 혜안(hyean.ai) 진행 현황

한국형 예측 마켓 + 예측 커뮤니티 서비스의 개발 진행 상황입니다.

## 🆕 이번 주 업데이트 (2026-06-09)

- **🔐 결과검증(Audit) 모듈 신규 구축 — 서비스 핵심 IP**
  예측 결과를 **외부 공신력 있는 출처로 인증**하는 검증 절차를 도입했습니다.
  · **검증관(전문 등급)** 이 결과와 외부 출처(언론·공식 발표 등)를 함께 제출
  · **서로 다른 검증관 2인의 독립 합의**가 모여야 정산이 실행됨(교차검증)
  · 검증 출처는 공개되며, 정산 완료 화면에 **"외부 출처 N건으로 검증됨"** 으로 표기 → 신뢰·투명성 확보.
- **캠페인 상세 페이지 고도화** (칼시·폴리마켓 스타일) — 총 거래량·참여자·베팅 수·마감 D-day **통계 바**, **실시간 활동 피드**("○○님이 YES에 289냥"), **정산 결과 + 검증 출처** 배너.
- **메인/디자인 고급화** — 칼시·폴리마켓식 마켓 그리드, 상단 네비 + 카테고리 바, 라이브 티커 등 **동적 연출**, 전체 **모바일 반응형**.
- **소셜 로그인 — X(트위터) 연동 완료**, 카카오 준비 중. (이메일 로그인과 함께 제공)
- **이용약관 · 개인정보처리방침 · 공통 푸터** 추가, **사용안내(게임 규칙)** 페이지 신설.

## 🛠 기술 스택 · 아키텍처 (요약)

- **프론트엔드/풀스택**: Next.js 16 (App Router · 서버 컴포넌트 · TypeScript), 서버사이드 렌더링(SSR), 폰트 Pretendard
- **백엔드 / DB**: Supabase (PostgreSQL, 서울 리전) — 인증(Auth) · 데이터 · 실시간
- **포인트 정합성**: 냥은 **단일 원장(append-only ledger)** 이 진실원천, 잔액은 DB 트리거로 자동 동기화 → 위변조·오류 추적 가능
- **보안 모델**: 행 단위 접근제어(RLS) + 베팅·정산·포인트 지급은 **서버 정의 함수(SECURITY DEFINER RPC)** 로만 수행 → 클라이언트 임의 조작 차단
- **정산 방식**: **파리뮤추얼**(승자에게 패자 풀을 베팅 비례로 배분), 수수료·판정 기준 사전 공시
- **배포 / 운영**: GitHub → Vercel 자동 배포, DB 변경은 마이그레이션으로 버전 관리, 코드+문서를 단일 저장소로 관리
